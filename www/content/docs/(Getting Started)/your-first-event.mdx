---
title: Your First Event
description: Track a real typed event with a tiny console provider before wiring a vendor.
---

This tutorial uses a local console provider so you can see the exact event object immediately. After that, swap in PostHog, Bento, Pirsch, Visitors, Proxy, or any custom provider without changing the call site.

## 1. Install

```bash
pnpm install trakoo
```

## 2. Define one event

```typescript title="lib/events.ts"
import type { CreateEventDefinition, EventCollection } from 'trakoo';

export const appEvents = {
  buttonClicked: {
    name: 'button_clicked',
    category: 'engagement',
    properties: {} as {
      buttonId: string;
      location: 'hero' | 'pricing' | 'nav';
    }
  }
} as const satisfies EventCollection<Record<string, CreateEventDefinition<string>>>;

export type AppEvents = typeof appEvents;
```

The event key is `buttonClicked`, but the tracked event name is `button_clicked`.

## 3. Create a console provider

```typescript title="lib/analytics.ts"
import {
  BaseAnalyticsProvider,
  createClientAnalytics,
  type BaseEvent,
  type EventContext
} from 'trakoo/client';
import type { AppEvents } from './events';

class ConsoleProvider extends BaseAnalyticsProvider {
  name = 'ConsoleProvider';

  initialize() {
    this.log('ready');
  }

  track(event: BaseEvent, context?: EventContext) {
    console.log('tracked', { event, context });
  }

  identify(userId: string, traits?: Record<string, unknown>) {
    console.log('identified', { userId, traits });
  }

  pageView(properties?: Record<string, unknown>, context?: EventContext) {
    console.log('page view', { properties, context });
  }

  pageLeave(properties?: Record<string, unknown>, context?: EventContext) {
    console.log('page leave', { properties, context });
  }

  reset() {
    console.log('reset');
  }
}

export const analytics = createClientAnalytics<AppEvents>({
  providers: [new ConsoleProvider({ debug: true })],
  debug: true
});
```

## 4. Track it

<Tabs>
<Tab title="React">

```tsx title="components/SignupButton.tsx"
import { analytics } from '@/lib/analytics';

export function SignupButton() {
  return (
    <button
      onClick={() => {
        analytics.track('button_clicked', {
          buttonId: 'signup-cta',
          location: 'hero'
        });
      }}
    >
      Sign up
    </button>
  );
}
```

</Tab>
<Tab title="Svelte">

```svelte title="SignupButton.svelte"
<script lang="ts">

<button on:click={trackSignupClick}>Sign up</button>
```

You should see an event like this in the browser console:

```typescript
{
  event: {
    category: 'engagement',
    action: 'button_clicked',
    properties: {
      buttonId: 'signup-cta',
      location: 'hero'
    }
  }
}
```

## 5. Replace the console provider

```bash
pnpm install posthog-js
```

```typescript title="lib/analytics.ts"
import { createClientAnalytics } from 'trakoo/client';
import { PostHogClientProvider } from 'trakoo/providers/client';
import type { AppEvents } from './events';

export const analytics = createClientAnalytics<AppEvents>({
  providers: [
    new PostHogClientProvider({
      token: import.meta.env.VITE_POSTHOG_KEY
    })
  ]
});
```

Your component still calls the same event:

```typescript
analytics.track('button_clicked', {
  buttonId: 'signup-cta',
  location: 'hero'
});
```
