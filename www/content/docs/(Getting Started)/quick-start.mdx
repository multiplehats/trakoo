This path gets you to a working, typed analytics setup with a real provider. It uses PostHog in the first example because it supports both browser and server tracking, but the same event definitions work with every provider.

## 1. Install

```bash
pnpm install trakoo posthog-js posthog-node
```

Use npm, yarn, or bun if that is what your project uses. See [Installation](/docs/installation) for provider-specific packages.

## 2. Define events once

```typescript title="lib/events.ts"
import type { CreateEventDefinition, EventCollection } from 'trakoo';

export const appEvents = {
  buttonClicked: {
    name: 'button_clicked',
    category: 'engagement',
    properties: {} as {
      buttonId: string;
      location: 'hero' | 'nav' | 'pricing';
    }
  },
  userSignedUp: {
    name: 'user_signed_up',
    category: 'user',
    properties: {} as {
      email: string;
      plan: 'free' | 'pro' | 'enterprise';
      referralSource?: string;
    }
  }
} as const satisfies EventCollection<Record<string, CreateEventDefinition<string>>>;

export type AppEvents = typeof appEvents;
```

The object keys are for organizing your code. The `name` values are the event names you pass to `track()`.

## 3. Create browser analytics

```typescript title="lib/analytics.ts"
import { createClientAnalytics } from 'trakoo/client';
import { PostHogClientProvider } from 'trakoo/providers/client';
import type { AppEvents } from './events';

export const analytics = createClientAnalytics<AppEvents>({
  providers: [
    new PostHogClientProvider({
      token: import.meta.env.VITE_POSTHOG_KEY,
      api_host: import.meta.env.VITE_POSTHOG_HOST
    })
  ],
  debug: import.meta.env.DEV
});
```

`createClientAnalytics()` auto-initializes in the background. You can still call `await analytics.initialize()` yourself if your framework has a dedicated bootstrap point.

## 4. Track in your UI

```tsx title="components/SignupButton.tsx"
import { analytics } from '@/lib/analytics';

export function SignupButton() {
  return (
    <button
      onClick={() => {
        analytics.track('button_clicked', {
          buttonId: 'signup-cta',
          location: 'hero'
        });
      }}
    >
      Sign up
    </button>
  );
}
```

If you type `button_clickd` or forget `location`, TypeScript reports it before the event ships.

## 5. Identify users when you know them

```typescript
analytics.identify('user_123', {
  email: 'ada@example.com',
  plan: 'pro'
});

analytics.track('user_signed_up', {
  email: 'ada@example.com',
  plan: 'pro',
  referralSource: 'homepage'
});
```

The browser analytics instance is stateful: after `identify()`, later events include the current user context until you call `reset()`.

## 6. Add server analytics for critical events

```typescript title="lib/server-analytics.ts"
import { createServerAnalytics } from 'trakoo/server';
import { PostHogServerProvider } from 'trakoo/providers/server';
import type { AppEvents } from './events';

export const serverAnalytics = createServerAnalytics<AppEvents>({
  providers: [
    new PostHogServerProvider({
      apiKey: process.env.POSTHOG_API_KEY!,
      host: process.env.POSTHOG_HOST
    })
  ]
});
```

```typescript title="app/api/signup/route.ts"
import { serverAnalytics } from '@/lib/server-analytics';

export async function POST(request: Request) {
  const user = await createUser(await request.json());

  await serverAnalytics.track('user_signed_up', {
    email: user.email,
    plan: user.plan
  }, {
    userId: user.id,
    user: {
      email: user.email,
      traits: { plan: user.plan }
    }
  });

  await serverAnalytics.shutdown();

  return Response.json({ ok: true });
}
```

Server analytics is stateless: pass user context with each event. In serverless environments, call `shutdown()` before the function exits so providers can flush.

## Switch or combine providers

One setup can send events to several services:

```typescript
import {
  BentoClientProvider,
  PostHogClientProvider,
  VisitorsClientProvider
} from 'trakoo/providers/client';

export const analytics = createClientAnalytics<AppEvents>({
  providers: [
    new PostHogClientProvider({ token: import.meta.env.VITE_POSTHOG_KEY }),
    {
      provider: new BentoClientProvider({
        siteUuid: import.meta.env.VITE_BENTO_SITE_UUID
      }),
      methods: ['identify', 'track']
    },
    new VisitorsClientProvider({
      token: import.meta.env.VITE_VISITORS_TOKEN
    })
  ]
});
```

Provider routing lets you keep each service focused on the events it can handle well.
