## The aha moment

One event definition becomes autocomplete, validation, and provider routing everywhere you track it.

```typescript title="lib/events.ts"
import type { CreateEventDefinition, EventCollection } from 'trakoo';

export const appEvents = {
  userSignedUp: {
    name: 'user_signed_up',
    category: 'user',
    properties: {} as {
      email: string;
      plan: 'free' | 'pro' | 'enterprise';
    }
  }
} as const satisfies EventCollection<Record<string, CreateEventDefinition<string>>>;

export type AppEvents = typeof appEvents;
```

```typescript title="lib/analytics.ts"
import { createClientAnalytics } from 'trakoo/client';
import {
  PostHogClientProvider,
  VisitorsClientProvider
} from 'trakoo/providers/client';
import type { AppEvents } from './events';

export const analytics = createClientAnalytics<AppEvents>({
  providers: [
    new PostHogClientProvider({
      token: import.meta.env.VITE_POSTHOG_KEY
    }),
    new VisitorsClientProvider({
      token: import.meta.env.VITE_VISITORS_TOKEN
    })
  ]
});
```

```typescript
analytics.track('user_signed_up', {
  email: 'ada@example.com',
  plan: 'pro'
});
```

TypeScript now knows the valid event names and required properties. If you add Bento for lifecycle email or move browser events through the Proxy provider later, the application code that calls `analytics.track()` does not change.

## Start fast

<Steps>
<Step>

### Install trakoo

```bash
pnpm install trakoo
```

</Step>
<Step>

### Add the SDKs for your providers

```bash
pnpm install posthog-js posthog-node
pnpm install @bentonow/bento-node-sdk
pnpm install @emitkit/js
```

Some client providers load their script from a CDN at runtime. See [Installation](/docs/installation) for the exact matrix.

</Step>
<Step>

### Track your first event

```typescript
analytics.identify('user_123', {
  email: 'ada@example.com',
  plan: 'pro'
});

analytics.track('user_signed_up', {
  email: 'ada@example.com',
  plan: 'pro'
});
```

</Step>
</Steps>
